§ 3CONTEXT CROPPING
BLOCK keeps a fixed recent depth and drops the middle in turn-blocks. STANDARD CULLING lets the oldest messages peel off one at a time as usage reaches the fall-off line.

Continuous rolling buffer — no fixed recent block. The system prompt and the first two exchanges are always preserved; whole tool-call rounds fall off together.
◢ CONTEXT CROPPING · The full conversation is always kept in this terminal — nothing is ever deleted from the chat; only the model's view is trimmed. Two methods:
▸ ROLLING BLOCK — once usage crosses the cutoff (context × threshold%), the oldest middle turns flutter off while the most-recent Context Depth pairs are always held.
▸ STANDARD CULLING — a continuous rolling buffer: the oldest messages peel off one at a time the moment usage reaches the fall-off line (context × fall-off%). No fixed recent block — history slides off naturally as it ages.
Both always preserve the system prompt and the first two back-and-forth exchanges (anchoring the original task), and both drop tool-call / tool-result rounds whole so the trace never orphans a tool_call_id. Indicator: CTX meter.
§ 4SYSTEM PROMPT / TEMPLATE
API PROVIDED sends the structured message array and lets the endpoint apply the model's own chat template (default, works everywhere). QWEN3 and GEMMA render the exact prompt string here — including special/channel tokens, thinking and tool-call structure — and post it to /completions so it reaches the model byte-for-byte. Use these when the server has no template for your model or applies the wrong one.
Display only — relabels assistant turns in this terminal. Never sent to the model; leave blank for the default [ASSI] tag.
Dynamic token: <time_local> is replaced — on every send — with the system's current date & time as MONTH, DAY, YEAR, HH:MM AM/PM (e.g. June, 25, 2026, 3:07 PM). Drop it anywhere in the prompt so the model always knows "now" — including when a scheduled job fires.

§ 9NETWORK
When enabled, the terminal binds to all interfaces so other machines on the local network can open the URL. Every connected browser (this machine, other PCs, secondary monitors) sees the same chat and settings in real time via the sync bus.
RESTART REQUIRED after toggling. Always live: SSE event bus keeps connected sessions mirrored.
