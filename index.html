§ 3CONTEXT & ROLLING WINDOW
Cuts at — tok (context × threshold).
Most-recent message pairs always kept in the model's window.
◢ ROLLING CONTEXT WINDOW · The full conversation is always kept in this terminal — nothing is ever deleted from the chat. Only the model's view rolls: once usage crosses the cutoff (context × threshold%), the oldest middle turns "flutter off" and peel away as they age. Always preserved: the system prompt, the first two back-and-forth exchanges (anchoring the original task), and the most-recent Context Depth pairs. Widening the context window or threshold re-admits older turns. Tool-call / tool-result rounds stay whole so the trace never orphans a tool_call_id. Indicator: CTX meter.

§ 8NETWORK
When enabled, the terminal binds to all interfaces so other machines on the local network can open the URL. Every connected browser (this machine, other PCs, secondary monitors) sees the same chat and settings in real time via the sync bus.
RESTART REQUIRED after toggling. Always live: SSE event bus keeps connected sessions mirrored.
