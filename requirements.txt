# CHATSGI / TERMINAL — Section 9 · rev 2.0
#
# The backend (start.py) is intentionally implemented using the Python ≥ 3.10
# standard library only — no third-party packages are required to run it.
#
# Stdlib modules used (NOT installable from PyPI — they ship with CPython):
#   argparse, json, os, queue, secrets, shutil, signal, socket, subprocess,
#   sys, threading, time, traceback, urllib (request/parse/error),
#   http.server, socketserver, pathlib
#
# To run:
#     python start.py
#
# Verify Python version (3.10 or newer):
#     python -V
#
# If a future feature ever needs a real PyPI package, list it below with a
# pinned version. Do NOT add fake or hallucinated package names here.
#
# ── Optional: Piper neural text-to-speech ─────────────────────────────────
# Only required to use Settings → § 7 SPEECH → TTS Engine → "PIPER TTS".
# The villager-talk blip SFX engine and the entire rest of the terminal still
# run on the stdlib alone, so this stays OPTIONAL. The backend imports piper
# lazily and degrades gracefully (reporting "not installed") when it is absent.
#
#     pip install piper-tts
#
# The synthesis wrapper in start.py supports the 1.2.x (synthesize_stream_raw)
# and 1.3+ (synthesize_wav) APIs, so a newer release also works.
piper-tts==1.2.0

# ── Browser tool: local, self-hosted Playwright ───────────────────────────
# Powers Settings → BROWSER and the in-terminal browser MCP server
# (browser_mcp_server.py) — a real, headed Chromium driven locally with NO
# npx, NO Docker and NO external Playwright server. The Chromium engine itself
# is downloaded into a project-local ./.browser/ directory.
#
# Don't `pip install` this by hand — run the dedicated setup helper, which
# installs the package AND the Chromium engine (and is what the UI's
# "INSTALL ENGINE" button / the backend auto-installer call):
#
#     python browser_setup.py
#
# Left unpinned so `playwright install chromium` always fetches the matching
# engine build (the package and its browser ship in lock-step).
playwright>=1.40
